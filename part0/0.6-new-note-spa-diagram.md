# Exercise 0.6: New Note in Single Page App Diagram

Sequence diagram showing what happens when the user creates a new note using the single-page version of the notes app at
https://studies.cs.helsinki.fi/exampleapp/spa

```mermaid
sequenceDiagram
    participant browser
    participant server

    Note right of browser: User types a note in the text field and clicks the "Save" button

    Note right of browser: JavaScript intercepts the form submit event (event.preventDefault()).<br/>The page does NOT reload. The new note object is created in the browser.

    Note right of browser: The browser immediately adds the new note to the notes list<br/>and re-renders the UI — all done locally without contacting the server.

    browser->>server: POST https://studies.cs.helsinki.fi/exampleapp/new_note_spa
    Note right of browser: The note is sent as JSON in the request body:<br/>{ "content": "User's new note", "date": "2024-06-22" }
    activate server
    server-->>browser: HTTP 201 Created
    Note left of server: Server saves the new note.<br/>Responds with status 201 — no redirect needed.
    deactivate server

    Note right of browser: The browser receives the 201 response and confirms the note was saved.<br/>The page was never reloaded — everything was handled by JavaScript.
```
